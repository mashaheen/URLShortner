Steps needed from `url-shortener-plan.md` to implement the public URL-shortening page:

1. Create the frontend folder:
   - `public-page/`

2. Create the public page file:
   - `public-page/index.html`

3. Make it a standalone static HTML page:
   - Plain HTML
   - CSS
   - Vanilla JavaScript
   - No frontend framework
   - Can be opened directly in the browser

4. Use this design direction:
   - Dark background
   - Minimal typographic style
   - Single centered card

5. Add these UI elements:
   - Page title: `Paste your URL here`
   - Full-width text input
   - Input placeholder: `https://your-long-url.com/...`
   - `Shorten` button
   - Hidden result area
   - Clickable generated short URL
   - `Copy` button
   - Error message area for invalid URLs

6. Add frontend configuration:
   ```js
   const API_BASE = "http://localhost:3000";
   ```

7. Implement behavior with Vanilla JS:
   - On `Shorten` button click, read the input value
   - Also submit when the user presses `Enter`
   - Validate that the URL starts with `http://` or `https://`
   - Show an inline error if validation fails
   - Send a POST request to:
     ```js
     `${API_BASE}/shorten`
     ```
   - Request body:
     ```json
     {
       "originalUrl": "https://example.com"
     }
     ```
   - Include header:
     ```http
     Content-Type: application/json
     ```

8. Handle API response:
   - On success, show the returned `shortUrl`
   - Display it as a clickable link
   - On failure, show the API error message

9. Implement copy behavior:
   - Use:
     ```js
     navigator.clipboard.writeText(shortUrl)
     ```
   - Change the button label to `Copied!`

10. Backend requirements this page depends on:
   - `POST /shorten` route must exist
   - `createShortUrl` controller must return:
     ```json
     {
       "shortUrl": "http://localhost:3000/abc1234",
       "shortCode": "abc1234"
     }
     ```
   - `cors()` must be enabled in `backend/server.js` so the static page can call the API.
